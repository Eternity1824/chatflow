# ChatFlow — DynamoDB Schema (Assignment 3)

## 1. Canonical table — `messages_by_id`

Single-writer table.  Every chat message is written here exactly once by
`consumer-v3` using a conditional PutItem.

| Attribute     | Type   | Role               | Notes |
|---|---|---|---|
| `messageId`   | String | **Partition key**  | UUID generated by server-v2, globally unique |
| `roomId`      | String | Attribute          | Room the message belongs to |
| `userId`      | String | Attribute          | Sender user ID |
| `username`    | String | Attribute          | Display name at time of send |
| `message`     | String | Attribute          | Message body |
| `timestamp`   | Number | Attribute          | Client-originated Unix epoch ms |
| `messageType` | String | Attribute          | TEXT / JOIN / LEAVE |
| `roomSequence`| Number | Attribute          | Per-room monotonic sequence (assigned by server-v2) |
| `ingestedAt`  | Number | Attribute          | Wall-clock Unix ms when DynamoDB write completed |
| `serverId`    | String | Attribute          | Which server-v2 instance published this message |

**Idempotency guard:**
```
ConditionExpression: attribute_not_exists(messageId)
```
If the condition fails, the item already exists → skip silently, ack normally.

**DynamoDB Streams:** enabled (`NEW_IMAGE`), feeds Lambda A (`CdcProjectorHandler`).

## 2. Projection table — `room_messages`

Read model: retrieve recent messages in a room ordered by sequence.

| Attribute      | Type   | Role               |
|---|---|---|
| `roomId`       | String | **Partition key**  |
| `roomSequence` | Number | **Sort key**       |
| `messageId`    | String | Attribute (FK ref) |
| `userId`       | String | Attribute          |
| `username`     | String | Attribute          |
| `message`      | String | Attribute          |
| `timestamp`    | Number | Attribute          |
| `messageType`  | String | Attribute          |
| `projectedAt`  | Number | Attribute — CDC write time (for lag measurement) |

Query pattern: `PK=roomId, SK BETWEEN seq_start AND seq_end` (paginated).

---

## 3. Projection table — `user_messages`

Read model: retrieve messages sent by a specific user.

| Attribute      | Type   | Role               |
|---|---|---|
| `userId`       | String | **Partition key**  |
| `messageId`    | String | **Sort key**       |
| `roomId`       | String | Attribute          |
| `timestamp`    | Number | Attribute          |
| `message`      | String | Attribute          |
| `projectedAt`  | Number | Attribute          |

Query pattern: `PK=userId` with optional `SK` range / `timestamp` filter via GSI.

---

## 4. Projection table — `user_rooms`

Read model: which rooms a user has been active in.

| Attribute       | Type   | Role               |
|---|---|---|
| `userId`        | String | **Partition key**  |
| `roomId`        | String | **Sort key**       |
| `lastSeenAt`    | Number | Attribute — epoch ms of most recent message in this room |
| `messageCount`  | Number | Attribute — running total (updated by Lambda A with ADD expression) |
| `projectedAt`   | Number | Attribute          |

Write pattern (Lambda A): `UpdateItem` with
```
SET lastSeenAt = :ts, projectedAt = :now  ADD messageCount :one
ConditionExpression: attribute_exists(userId)  (or unconditional for first entry)
```

---

## 5. ingestedAt & projection lag

```
ingestedAt   = time consumer-v3 confirmed the DynamoDB PutItem succeeded
projectedAt  = time Lambda A wrote the projection record

projection_lag_ms = projectedAt − ingestedAt
```

Target: p99 projection lag < 2 s under normal load.

---

## 6. Redis / ElastiCache (analytics)

| Key pattern                   | Type    | Semantics |
|---|---|---|
| `room:{roomId}:msg_count`     | String  | INCR per message (room total) |
| `user:{userId}:msg_count`     | String  | INCR per message (user total) |
| `room:{roomId}:active_users`  | HyperLogLog | PFADD userId |
| `leaderboard:rooms`           | Sorted Set  | ZADD roomId score=msg_count |

Updated by Lambda B after Lambda A has already written the DynamoDB projections
and published analytics events to SQS.
TTL: none on counters; leaderboard sorted sets expire after 24 h (configurable).
